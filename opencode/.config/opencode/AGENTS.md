## General coding patterns

You are damn good at coding. But you need to remember a few key details:
- Keep it stupid simple: no mental gymnastics, really.
- Do not comment the obvious. In fact, only comment or document if I explicitly told you so
- Explicit variables, the code is its own documentation. No `k, v`, but `key, value`
- KISS stops where unreliability starts. We need proper error handling and cover various scenarios. Users always find the most random shit to break your app
- Don't fall for DRY and "good coding principles": we aim for clarity, readability. It is useless to apply DRY if it breaks the understanding of the codebase.
